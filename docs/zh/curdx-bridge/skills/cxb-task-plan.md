# cxb-task-plan

从确认的方案生成可执行的任务制品。

## 概述

`cxb-task-plan` 连接高层方案和可执行步骤。它先运行完整的协作规划流程（[`cxb-plan`](/zh/curdx-bridge/skills/cxb-plan)），然后将批准的方案转化为结构化的任务制品，供 [`cxb-task-run`](/zh/curdx-bridge/skills/cxb-task-run) 执行。

## 工作流

### 1. 初始化

- 接收你的需求
- 分析项目技术栈和结构
- 必要时研究不熟悉的技术

### 2. 协作规划

调用 [`cxb-plan`](/zh/curdx-bridge/skills/cxb-plan) 运行完整设计流程：
- 需求澄清（五维就绪度模型）
- 灵感发散
- 方案撰写
- 评分审查

### 3. 用户确认

展示方案摘要供你确认：
- 目标声明
- 实施步骤
- 验收标准

### 4. 生成任务制品

在 `.curdx/` 下创建三个文件：

| 文件 | 用途 |
|------|------|
| `state.json` | 执行状态 — 当前步骤、尝试次数、状态 |
| `todo.md` | 带状态标记的任务列表，用于进度追踪 |
| `plan_log.md` | 执行日志，记录决策和历史 |

### 5. 开始执行

可选启动自动循环守护进程，通过 [`cxb-task-run`](/zh/curdx-bridge/skills/cxb-task-run) 立即开始执行步骤。

## 设计原则

- **协作设计** — 使用全部三个角色（设计师、灵感源、评审员）
- **粗粒度步骤** — 仅定义步骤标题，详细设计在执行时进行
- **研究驱动** — 遇到不熟悉的技术先调研再规划
